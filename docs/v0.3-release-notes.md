# V0.3 Release Notes — Runnable Local Exchange

V0.3 converts the specification and domain foundation into a browser-operable vertical slice.

## Integrity changes

- Explicit evidence-to-claim bindings replace broad evidence arrays.
- Authority is checked against actor, organization, system, time, and exact committed payment hash.
- Multiple valid authorities for the same role produce ESCALATE rather than silent first-match selection.
- Policy versions must exist in the current policy registry.
- Deterministic decision fingerprints permit substantive replay comparison while receipt IDs remain unique.
- Negative bypass, replay, and duplicate findings require evidence bindings instead of caller booleans alone.
- AER content now includes purpose, scope, limitations, authority basis, policy version, history, and verification instructions.

## Application changes

- Next.js landing page, workspace, API routes, correction path, preservation path, and public registry page.
- Local durable JSON persistence for development demonstration only.
- End-to-end HOLD -> correction -> ALLOW -> AER -> registry flow.

## Non-production limitations

No production authentication, PostgreSQL runtime connection, Stripe charge, KMS/HSM signing, malware scanning, private evidence upload, email, backup service, or independent decision authority is activated in V0.3.
