# First-Release Screen Map

| # | Screen | Primary job | Required elements |
|---|---|---|---|
| 1 | Start / Sign in | Explain category and begin securely | One-minute explanation, login, boundary statement, privacy link |
| 2 | Organization setup | Create accountable entity | Legal/display name, accountable owner, identity level, role confirmation |
| 3 | System registration | Name system and purpose | Name, version, environment, intended purpose, owner |
| 4 | Route definition | Bound one consequential action | Initiator, action, threshold, destination, policy, expected outcome |
| 5 | Authority | Record who may authorize what | Issuer, actor, scope, dates, revocation, supporting reference |
| 6 | Evidence | Attach/reference proof | Class, source, timestamp, hash, validity, custody, visibility |
| 7 | Test result | Explain exact state | Result, failed IDs, reasons, evidence considered, next action |
| 8 | Correction | Fix without payment | Checklist, upload/reference, version delta, protected rerun |
| 9 | AER preview | Show bounded record | Identity, scope, evidence index, tests, history, boundary |
| 10 | Durable record checkout | Offer preservation, not approval | Exact promise, exclusions, price, terms, receipt |
| 11 | Record confirmation | Prove issuance | TA14-RID, timestamp, manifest hash, exports, lookup link |
| 12 | Registry lookup | Enable third-party reliance | Status, scope, version, limitations, signature, history, misuse report |
| 13 | Account history | Maintain records | Systems, routes, versions, payments, exports, material changes |
| 14 | Admin control | Protect integrity | Misuse queue, suspension, revocation, audit trail, no silent edits |

## Required empty/error states
Every screen must define: no-data state, unauthorized state, validation failure, stale-version conflict, and recoverable platform failure.
