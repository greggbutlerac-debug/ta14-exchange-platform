# V0.2 Upgrade Notes

## Completed
- Canonical JSON serialization and deterministic SHA-256 input manifests.
- Ed25519 receipt signing and independent signature verification.
- Explicit route/version, policy version, requirement-library version, signing-key ID, timestamp, next action, and evidence references in receipts.
- Strict timestamp and authority-validity checks.
- `OUT_OF_SCOPE` classification for transactions outside the selected template rather than a misleading HOLD.
- Signed AER manifest generator with tamper verification.
- Database hardening migration with validity constraints, signing fields, forced RLS, and initial tenant policies.
- Expanded automated tests for precedence, correction conditions, malformed dates, canonicalization, signatures, AER integrity, and tampering.
- GitHub Actions CI verification workflow.
