# V0.3.1 Integrity Correction

V0.3.1 corrects the truthfulness and continuity boundaries of the local vertical slice.

## Completed

- Preserves the original HOLD and every corrected route version instead of overwriting history.
- Accepts user-entered procurement authority, finance authority, and beneficiary evidence records.
- Generates AER history from stored events rather than reconstructing it from the latest receipt.
- Validates route-creation and correction payloads with Ajv.
- Uses a canonical structured payment commit with USD minor units.
- Labels fixture evidence for bypass, replay, duplicate-payment, execution, and outcome observations.
- Uses atomic file replacement, serialized writes, and optimistic version checks.
- Hash-chains route events and verifies the chain before AER issuance.
- Publishes event hashes and the development public key in a downloadable verification bundle.
- Displays both AER-signature validity and event-chain validity on the registry page.
- Adds storage continuity and version-conflict acceptance tests.

## Verification

- TypeScript: passed
- Automated tests: 18 passed across 6 test files
- Next.js production build: passed

## Boundary

The file-backed store and development signing key remain local demonstration mechanisms. PostgreSQL, authentication, managed keys, live payments, private evidence custody, and independent decision authority are deferred to V0.4.
